# 💻 Super Clip 实时开发留痕 (Development Log)

本文档基于持续集成的 `task.md` 完整自动记录，用于追踪项目的重大架构演进、UI/UX重构以及环境配置。这见证了项目从最初的基础剪贴板发展为兼具深度本地文件搜索、高性能 Rust 引擎和顶级玻璃拟物美学的**终极效率工具**。

---

## 📅 v0.2.3 (开源发布准备阶段)

这是将应用推向开源社区，极致精简体积与强化运行稳定性的最后阶段。

### Phase 57: 开源清理与仓库准备 (Open Source Preparation)
- 彻底移除了无用的冗余 AI 后端文件和 22 个调试日志废品，使代码库保持整洁。
- 清理了 `main.rs` 中的死亡函数和无用模块导入。
- 清理了 `Cargo.toml` 中多余的依赖（`reqwest`, `fastembed`, `tokio` 等 7 个不再支持的无用包）。
- 撰写了全新的中英混合高质量 `README.md`。

### Phase 56: 极致视觉打磨 (Logo Breathing Animation Polish)
- 重构了应用左上角的动态 Logo。移除了旧版的发散光晕，改为干净、轮廓分明的宝石质感灯带。
- 创新性地引入了 **分类动态相移** 算法。切换不同的分类标签时，Logo 会实时改变渐变色（Slate -> Emerald, Indigo, Pink等），并配合 GPU 加速的缓慢外里双层呼吸动画。

### Phase 52 & Phase 50: 功能回顾与原生化 (Remove Heavy AI & Revert to Word Stats)
- 将臃肿的大语言模型总结与归类（LLM）移除，转而回归轻量级的本地词汇统计方案。
- 重新利用 `Intl.Segmenter`（无需网络、纯原生 JavaScript）实现了高性能的全局内容高频词汇统计仪表盘。
- 增加了全新的“三小时内”精细化时间维度导航标签。

## 📅 历史迭代架构里程碑 (Architectural Milestones)

### 🚀 架构重构与性能探索 (Phases 1-6)
- **Rust 重写**：彻底抛弃了早期的 Python 侧载进程，使用纯 Rust 和 Tauri 重构了后端进程管理。
- **本地 OCR 引擎**：早期尝试集成了 Windows 原生 OCR API，后又研究了基于 `fastembed` 的向量化语义搜索和 `ort` (ONNX Runtime) 模型推理的无网离线功能。
- **稳定性极度增强**：实现全局恐慌 Hook 用于终端 Debug，以及对剪贴板变化进行线程安全的监听逻辑。

### 🎨 极致的 UI/UX 和仪态打磨 (Phases 7-14, 28-35)
- **Dashboard 交互式面板**：加入了基于时间的智能概览视图，包括基于径向图（Radial Donut Charts）的内容模块分类。
- **Hit-App 专业美学**：采用了高级的 **深色石墨黑 (Graphite)** 和 **蔚蓝电光 (Electric Blue)** 搭配。全量重构列表和背景为高级全局玻璃拟物（Glassmorphism）和透光微动效。
- **多窗口系统支持**：去掉了操作系统原生的标题栏，全量定制了支持拖拽功能的顶部可变色 TitleBar。
- **划选自动复制**：针对识别的图片文本加入了（Feishu / PixPin 风格的）文本隐形透叠点击一键复制系统。

### 🔍 "Everything" SDK 深度集成 (Phases 15-20)
- **极简模式突起**：加入了类似 macOS Spotlight 或 uTools 的 `Ctrl+M` 极简居中搜索界面。
- **底层互通**：将著名的 "Everything SDK" 通过 C-ABI 绑定到了 Rust 后台，在极简模式中可以即时双向索引本地所有文件。
- **过滤与直达**：实现了基于分类（文档、图片、可执行程序）的高级过滤器，支持方向键极速选择，更可通过回车直接用本地默认程序打开文件。

### 🤖 本地 AI 与自动化实验 (Phases 26-47) (历史尝试)
- 曾完整集成过本地 `llama-server` 后台侧载运行大语言模型。
- 实现了复杂的“后台大脑”提示词（如：高级生产力架构师/幕僚长角色），在零前台打扰的情况下自动在后台为用户的剪切板提取关键意图并打标签和图表分析。
- *(注：为了支持更轻量级的开源分发和保证基础搜索的速度，此部分“全功能超大体积模式”在最新稳定版中已被精简封存。)*
